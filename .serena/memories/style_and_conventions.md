# Style and conventions
- Skills are written as concise Markdown documents with YAML frontmatter.
- Skill names use kebab-case directories and `name` fields.
- Prefer portability across runtimes; describe behavior generically instead of binding to agent-specific tools unless intentionally runtime-specific.
- Keep instructions short but actionable; include explicit safeguards and confirmation steps where needed.
- Shell snippets target bash and should be copyable.
